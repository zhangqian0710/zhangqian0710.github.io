9.Linux下Job项目之定时任务

发表于 2019-11-05 | | 阅读次数:
字数统计: 552字

​

​ 平时工作中可能会写很多Job项目，用来处理或者审核头一天公司产品或者后台产生的各种数据，例如活动数据、订单数据等。下面讲介绍如何自己可以用一台服务器来帮助我们在凌晨时分，人不在办公室而可以完成Job项目的启动及保留日志。

【提前准备】

1.一台Liunx服务器（本人系统是Contos7）
2.一个写好的并打包jar包的java项目： XXX.jar（通常是job项目）
【开始定时任务配置】
一、定时任务环境准备
1.安装crontabs服务
yum install vixie-cron
yum install crontabs
2.查看crontab服务（定时任务服务）是否已设置为开机启动，执行命令：
ntsysv
3.加入开机自动启动:
chkconfig –level 35 crond on
4.也可以用设置开机自动启动crond服务:
chkconfig crond on
（crond是linux下用来周期性的执行某种任务或等待处理某些事件的一个守护进程，与windows下的计划任务类似，当安装完成操作系统后，默认会安装此服务工具，并且会自动启动crond进程，crond进程每分钟会定期检查是否有要执行的任务，如果有要执行的任务，则自动执行该任务。）

二、设置jar自动执行
1.usr/目录下建立dbtodb文件夹，在dbtodb文件夹下新建一个xxx.sh,脚本内容为：

#!/bin/bash
cd /opt/jobs/jobnameXXX/
java -jar -Xms1024m -Xmx1536m XXX.jar
2.对xxx.sh赋权限，具体操作如下：
chmod 777 xxx.sh
3.由于Windows于linux编码格式不同，会出现（执行过程中出现/bin/bash^M: 坏的解释器：）需要对.sh进行编码转换
1. dos2unix dbtodb.sh
dos2unix: converting file dbtodb.sh to UNIX format …
2. sed -i ‘s/\r$//‘ xxx.sh
4.在控制台上输入以下命令
crontab -e
按i键编辑这个文本文件
0 1 * * * /usr/dbtodb/XXX.sh &>/usr/dbtodb/log.txt

（每天凌晨一点）
5.service crond start//启动服务
service crond stop//关闭服务
service crond restart//重新载入服务
service crond status//查看状态
